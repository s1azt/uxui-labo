2026.1 - 2026.2 • UXUIユニット

はじめに

UXUIユニットでは、社内のUXUI活動を活性化し、より良いプロダクト開発を推進するため、様々なイベントを企画しています。

このページでは、2026年1月〜2月に開催予定のイベントをご紹介します。
実践的なFigma研修から、気軽に参加できる座談会まで、多様な学びの機会をご用意しています。

2

座談会（UX情報発信）

UXUIに関する知見を共有し、組織全体のデザイン意識を高めるための座談会を企画しています。

以下、7つのテーマ案から、最適な形式を選定して開催予定です。

開催形式の検討

•

座談会形式：テーマごとにUXUIを実施したことのある他部署・他チームメンバーを選出（UX/UIに興味がある初心者が混ざっていても良い）

•

スピーカー形式：一旦UXUIユニットのみとする場合、［生成AI活用事例共有会］のようにスピーカー形式で実施

テーマ案

案1

UXUI社内取り組み事例共有会

社内で実際に起きた「良い変化」にスポットライトを当て、ポジティブな情報を展開するための施策

目的
  • • 成功パターンの横展開：「あのチームのやり方は、うちでも使える」という再現性のある知見を広める
  • • モチベーション向上：「うちの会社でも、こだわればここまで良いものが作れるんだ」という認識を持たせる
ポイント
  • • 「Before」と「思考プロセス」を重視
  • • 「どんな課題があり」「どう悩み」「なぜそのデザインに決めたか」という過程の展開
  • • 大規模リニューアルだけでなく、「ボタンの文言を変えたら問い合わせが減った」といった地味だが確実な改善で生まれた小さな成果こそ、開発メンバーが真似しやすい
ターゲット
  • • 自チームでの改善の糸口を探しているPM、リーダー層
  • • 「UXって言われても具体的に何をすればいいか分からない」と感じているエンジニア
案2

UXUI失敗談共有会

あえて「うまくいかなかった話」を共有することで、開発組織の風通しを良くし、UX活動への心理的ハードルを下げる施策
※案1とセットで失敗談→成功例としてつなげる

目的
  • • 心理的安全性の醸成：「UXUIの活動において、トライ＆エラーは当たり前」「失敗しても大丈夫」という文化を作る
  • • アンチパターンの学習：「ユーザーの声を無視して機能を追加したら、誰も使わなかった」といった同じ轍（てつ）を踏まないための防御策を学ぶ
ポイント
  • • 明るい雰囲気で：反省会のような重い空気ではなく、笑い飛ばせるようにポップな雰囲気を目指す
  • • 個人ではなくプロセスにフォーカス：「誰がミスったか」ではなく、「どの段階でユーザー確認をしておけば防げたか」というプロセスの振り返りを行い、最後は「教訓」で締める
ターゲット
  • • 失敗を恐れて手堅い（無難な）仕様に倒しがちなエンジニアやPM
  • • 「完璧なものを出さなきゃ」とプレッシャーを感じている若手メンバー
案3

UIクリニック（公開相談会）

開発中の画面や、「なんとなく使いにくいけど直し方がわからない」既存画面をスクリーンに映し、UXUIデザイナー（UXUIユニットメンバー）と開発者がその場で改善案をディスカッションする

目的

具体的な困りごとの解決と、デザイナー（UXUIユニットメンバー）の思考プロセス（なぜその修正が必要か）の共有

ポイント
  • • 「ダメ出し」ではなく「治療（クリニック）」というポジティブなスタンスで行う
  • • Before/Afterがその場で見えるため、納得感が高い
ターゲット

具体的な実装に悩むエンジニア、仕様を決めるPM
※実案件にて悩んだ・悩んでいるPJTがあれば、協力してもらうとよい

案4

「推しUI / 惜しいUI」持ち寄り座談会

自社プロダクトではなく、世の中にある普段使っているアプリやWebサービスについて、「ここの操作感が気持ちいい（推し）」「ここが使いづらくてイライラする（惜しい）」を語り合うライトな会

目的
  • • ユーザー視点を取り戻す
  • • 自社製品だと角が立つ批評も、他社製品なら気楽に分析でき、言語化のトレーニングになる
ポイント
  • • 「なぜ使いやすいのか？」の法則性を最後にデザイナーが解説すると、学びが深まる
  • • 自分だったらどういうUIにするのに！という意見を考えてもらう（言語化）
ターゲット

全職種（新卒や若手も参加しやすい）

案5

ヒューリスティック分析的な討論

感じている良いUI、悪いUIの意見を言い合う
（案4とかぶる）

案6

UI/UXに興味を持ったきっかけ

経験者・初心者それぞれの視点から語り合う

経験者: 自発的に課題を感じ、変えていこうとした話をする
初心者: なぜ興味を持ったのかと、YSDのUX/UIについて感じること
案7

Figma、Figjam利用の楽しさ、面白さ、可能性について

ツールの魅力を語り合い、活用の幅を広げる

おわりに

これらのイベントは、UXUIの知見を共有し、
組織全体のデザイン意識を高めることを目的としています。

研修で基礎を学び、座談会で実践的な知見を交換する。
この2つのアプローチで、YSDのUXUI活動をさらに推進していきます。
ぜひ、お気軽にご参加ください！
